# 產品規格：Agent Office Control Room

## 1. 使用者問題

當多個 AI Agent 同時工作時，使用者很難快速回答下列問題：

- 現在有哪些任務正在進行？
- 誰負責、做到哪裡、是否卡住？
- 哪些工作值得繼續消耗 token？
- 距離 context 或帳戶限制還有多遠？
- 哪個成果已經審查，可以放心使用？

本產品用具象的辦公室與角色動態，將抽象的 Agent 執行流程轉成可理解、可調度的團隊管理介面。

## 2. 核心畫面

### 辦公室視圖

- 俯視角辦公室場景。
- Agent 依工作狀態移動到資料室、分析區、製作區、審查室或等待區。
- 點擊 Agent 顯示任務、進度、最近動作、阻塞原因與用量。
- 拖曳任務卡到 Agent 或工作區以重新指派。

### 專案總覽

- 進行中、待核准、卡住、待審查與已完成任務。
- 專案目標、截止日、重要性及目前風險。
- 最近完成的成果與待使用者決策事項。

### Token 控制台

- 每個 Agent 的實際任務 token。
- 每個任務的預算、已用、預估完成成本。
- 目前 context 使用率與自動摘要狀態。
- 帳戶使用趨勢；無官方數據時必須標示為估算。
- 預警門檻與系統採取的動作。

## 3. 任務狀態

`Inbox → Planned → Assigned → Working → Waiting → Review → Done`

任務必備欄位：

- 標題與預期成果
- 專案與負責 Agent
- 優先級與截止日
- 執行狀態與完成比例
- Token 預算、實際用量與預估剩餘量
- 來源、輸出與驗收紀錄
- 阻塞原因與需要使用者核准的事項

## 4. Token 防焦慮機制

| 階段 | 建議門檻 | 系統行為 |
| --- | ---: | --- |
| 正常 | 0–59% | 正常執行 |
| 注意 | 60–74% | 顯示預警與完成成本預估 |
| 節流 | 75–84% | 輕量工作改用節省設定，限制新增 Agent |
| 保護 | 85–89% | 自動摘要、保存 checkpoint、停止非必要任務 |
| 臨界 | 90% 以上 | 暫停擴張工作並要求使用者決策 |

百分比必須標明基準：任務預算、context 容量或帳戶限制不可混為同一指標。

## 5. 權限原則

- 搜尋、讀取、分析與本地草稿可依任務權限自動執行。
- 發信、發布、交易、付費、刪除與對外修改必須取得明確核准。
- 每次核准需記錄目標、動作、時間與結果。

## 6. 初版成功標準

- 使用者在 10 秒內看懂有哪些 Agent 正在做什麼。
- 可以建立任務並拖曳指派給 Agent。
- 可以暫停、續跑與重新指派任務。
- 每個 Agent 顯示用量、進度與阻塞原因。
- iPad 橫向模式可完整操作主要功能。
- 模擬資料與真實 Codex 資料有明顯標示。
