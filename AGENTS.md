# Project Instructions

## Product intent

Build a calm, understandable control room for a multi-agent team. The interface should reduce anxiety and make work, cost, blockers, and approvals visible at a glance.

## Working conventions

- Keep the project manager as the single reporting interface to the user.
- Separate actual measured usage from estimates and plan-level limits.
- Prefer small, independently verifiable changes.
- Do not add an external service or paid dependency without documenting why it is needed.
- Design mobile and tablet layouts together with desktop layouts.
- Treat task data, prompts, sources, and usage history as private by default.
- Require explicit approval for publishing, messaging, deleting data, spending money, or changing external systems.

## Verification

- Run the relevant formatter, type checker, tests, and production build before marking implementation complete.
- Verify core interactions at desktop and tablet widths.
- Keep product decisions and unresolved questions in `docs/`.
