# Agent Office Control Room

一個以「可愛辦公室」呈現 AI 團隊運作的視覺化控制台。

使用者只需要對專案總管下達目標；專案總管會拆解任務、指派專業 Agent、整合結果，並持續回報進度、風險與 token／額度狀態。

## 線上原型

[開啟 Agent Room](https://agent-room.jaime2243.chatgpt.site)

目前版本以模擬資料展示完整操作流程：點選 Agent 查看工作、點任務後指派 Agent、建立新任務、暫停／送審，以及查看任務 token 預算。介面中的 token 並非帳戶真實剩餘額度。

## 核心目標

- 用辦公室場景呈現 Agent 的角色、位置與工作狀態。
- 支援拖曳任務卡進行指派與重新調度。
- 讓專案總管負責拆解、整合與對使用者報告。
- 顯示任務、Agent、專案三層 token 使用量與預算。
- 在接近額度或 context 上限前主動摘要、降級或暫停。
- 高風險外部操作必須經使用者核准。
- 可從 iPad、iPhone 與電腦查看進度。

## 初版 Agent

| Agent | 職責 |
| --- | --- |
| 專案總管 | 理解目標、拆解任務、分派工作、整合成果 |
| 資料研究員 | 搜尋來源、整理證據、標示可信度與缺口 |
| 分析員 | 執行財務、商業、資料與情境分析 |
| 執行員 | 製作程式、文件、表格與其他交付物 |
| 審查員 | 獨立檢查事實、計算、邏輯、風險與遺漏 |

## 產品原則

1. 使用者面對的是一個團隊，而不是一排聊天視窗。
2. 每個任務必須有負責人、狀態、預算與可驗收成果。
3. 真實用量、平台額度與系統推估必須清楚區分。
4. 研究與審查可以平行；多人同時修改同一成果應避免。
5. 所有 Agent 的中間噪音留在子執行緒，主線只保留決策與摘要。
